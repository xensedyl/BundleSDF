# BundleSDF 运行问题解决方法（2026-03-24）

## 1. 问题现象

运行命令：

```bash
python run_custom.py --mode run_video --video_dir /home/xense/BundleSDF/dataset/2022-11-18-15-10-24_milk --out_folder /home/xense/BundleSDF/result/bundlesdf_2022-11-18-15-10-24_milk --use_segmenter 1 --use_gui 1 --debug_level 2
```

出现过以下报错：

- `cv2.erode` / `cv2.resize` 报 `src is not a numpy array`
- `numpy` 报 `module 'numpy.core.multiarray' has no attribute 'unsignedinteger'`
- `RuntimeError: Unable to configure default ndarray.__repr__`
- 后续伴随 multiprocessing 的连锁错误（主进程崩溃后触发）

## 2. 根因

`bundlesdf` 环境中的 `numpy` 与 `opencv-python-headless` 安装状态被污染（残留多版本文件/元数据不一致），导致 OpenCV Python 绑定与 NumPy 运行时不兼容。

## 3. 已执行的修复

### 3.1 清理损坏的 site-packages 残留

```bash
rm -rf /home/xense/miniforge3/envs/bundlesdf/lib/python3.10/site-packages/numpy \
       /home/xense/miniforge3/envs/bundlesdf/lib/python3.10/site-packages/numpy.libs \
       /home/xense/miniforge3/envs/bundlesdf/lib/python3.10/site-packages/numpy-*.dist-info \
       /home/xense/miniforge3/envs/bundlesdf/lib/python3.10/site-packages/cv2 \
       /home/xense/miniforge3/envs/bundlesdf/lib/python3.10/site-packages/opencv_python_headless*.dist-info \
       /home/xense/miniforge3/envs/bundlesdf/lib/python3.10/site-packages/opencv_python_headless.libs
```

### 3.2 重新安装兼容版本

```bash
conda run --no-capture-output -n bundlesdf pip install --no-cache-dir --no-deps numpy==1.26.4 opencv-python-headless==4.10.0.84
```

### 3.3 代码层稳健性修复（已落地）

- `run_custom.py`
  - 增加 `normalize_mask`，统一掩码为单通道二值 `uint8`
  - 使用纯 NumPy 最近邻缩放 `resize_nearest`
  - 使用 `scipy.ndimage.binary_erosion` 作为腐蚀实现（避免依赖异常的 `cv2.erode`）
- `segmentation_utils.py`
  - `Segmenter.run()` 增加 mask 读取失败检查
  - 强制输出 contiguous 的单通道二值 `uint8`

## 4. 验证结果

在 `bundlesdf` 环境中验证通过：

```bash
python - <<'PY'

print("cv2", cv2.__version__)
a=np.zeros((8,8),dtype=np.uint8)

print(cv2.erode(a, np.ones((3,3),np.uint8)).shape)

PY
```

预期关键输出：

- `numpy 1.26.4`
- `cv2 4.10.0`
- `resize`/`erode` 可正常执行

## 5. 后续使用建议

- 固定在 `bundlesdf` 环境运行，避免混用系统 Python 与其他 venv。
- 避免在同一环境中反复混装 `conda` 与 `pip` 的核心底层包（尤其 `numpy/opencv/scipy`）。
- 如再次出现类似 ABI/绑定问题，优先采用“清目录 + 固定版本重装”方式处理。
